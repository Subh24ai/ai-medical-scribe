# Uploads Directory

This directory stores all uploaded and generated files.

## Directory Structure

```
uploads/
├── prescriptions/   - Generated prescription PDFs
├── documents/       - Patient medical documents
└── audio/          - Consultation audio recordings
```

## File Types

### Prescriptions
- Auto-generated by prescription service
- Format: PDF
- Naming: `prescription-{prescriptionNumber}.pdf`
- Example: `prescription-RX-2024-001.pdf`

### Documents
- Uploaded by doctors/patients
- Formats: PDF, JPG, PNG, JPEG
- Examples: Lab reports, X-rays, medical certificates

### Audio
- Recorded during consultations
- Formats: WebM, WAV, MP3
- Processed by AI transcription service
- Can be deleted after transcription (optional)

## Storage Options

### Local Storage (Development)
Files stored in these folders on the server.

### Cloud Storage (Production)
- AWS S3
- Azure Blob Storage
- Google Cloud Storage

Configure in `backend/src/config/config.js`:
```javascript
storage: {
  type: 'local', // or 's3', 'azure', 'gcs'
  localPath: './uploads'
}
```

## File Size Limits

Configured in server.js:
- Max file size: 50MB
- Adjustable based on needs

## Security

- Files are not publicly accessible
- Access requires authentication
- URLs are signed/temporary
- Virus scanning recommended for production

## Cleanup

Audio files can be deleted after transcription to save space:
```javascript
// In consultation service
if (transcription.success) {
  // Delete audio file
  fs.unlinkSync(audioPath);
}
```

## .gitignore

Add to `.gitignore`:
```
uploads/*
!uploads/.gitkeep
!uploads/*/gitkeep
!uploads/README.md
```

This keeps folder structure in git but ignores uploaded files.
